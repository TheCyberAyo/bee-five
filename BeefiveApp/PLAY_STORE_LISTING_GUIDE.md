Complete guide for creating your Play Store listing for Bee-Five.

### App Name
**Bee-Five**

### Short Description (80 characters max)
**Example**: "Strategic bee-themed board game with AI opponents and adventure mode"

### Full Description (4000 characters max)

**Template**:
```

✨ KEY FEATURES:
• Beautiful bee-themed design with smooth animations
• Multiple AI difficulty levels (Easy, Medium, Hard)
• Adventure mode with progressive challenges
• Local game progress tracking
• Intuitive touch controls
• Family-friendly gameplay suitable for all ages

🎯 PERFECT FOR:
• Strategy game lovers
• Board game enthusiasts
• Casual gamers looking for a challenge
• Families looking for fun together

Download Bee-Five now and start your strategic bee adventure!
```

### Category
- **Primary**: Games > Board
- **Secondary** (optional): Games > Strategy

### Content Rating
Complete the questionnaire. Expected rating: **Everyone**

### Target Audience
- **Age**: All ages
- **Content**: Family-friendly, no violence, no inappropriate content

## Required Graphics

### 1. App Icon
- **Size**: 512 x 512 pixels
- **Format**: PNG (32-bit with alpha channel)
- **Location**: Already in `android/app/src/main/res/mipmap-*/ic_launcher.png`
- **Requirements**:
  - No transparency
  - No rounded corners (Play Store adds them)
  - High quality, clear design

### 2. Feature Graphic
- **Size**: 1024 x 500 pixels
- **Format**: PNG or JPG
- **Purpose**: Banner at top of listing
- **Design Tips**:
  - Include app name "Bee-Five"
  - Show key features or gameplay
  - Use bee theme colors (yellow/black)
  - Keep text minimal and readable

### 3. Screenshots

#### Phone Screenshots (Required)
- **Minimum**: 2 screenshots
- **Maximum**: 8 screenshots
- **Aspect Ratio**: 16:9 or 9:16
- **Size**: 
  - Minimum: 320px (shortest side)
  - Maximum: 3840px (longest side)
- **Recommended**: 1080 x 1920 (9:16) or 1920 x 1080 (16:9)

**Suggested Screenshots**:
1. Main menu screen
2. Gameplay screenshot
3. AI game mode
4. Adventure map
5. Game in progress
6. Settings/options screen

#### Tablet Screenshots (Optional)
- Same requirements as phone
- Recommended if app is optimized for tablets

### 4. Promotional Video (Optional)
- **Format**: YouTube URL
- **Length**: 30 seconds to 2 minutes
- **Content**: Gameplay demonstration, features showcase

---

## Screenshot Guidelines

### Best Practices:
1. **Show Real Gameplay**: Use actual in-game screenshots
2. **Highlight Features**: Show different game modes
3. **Use Text Overlays Sparingly**: Only if necessary to explain features
4. **Consistent Style**: Use similar framing and style across all screenshots
5. **High Quality**: Ensure screenshots are clear and not pixelated
6. **Show Variety**: Different screens, modes, and features

### What to Include:
- ✅ Main menu
- ✅ Gameplay action
- ✅ Different game modes
- ✅ UI elements
- ✅ Game features

### What to Avoid:
- ❌ Too much text overlay
- ❌ Low resolution images
- ❌ Misleading screenshots
- ❌ Screenshots that don't match actual app

### Keywords

## Review Process

After submitting your listing:

1. **Initial Review**: 1-3 days
2. **Content Review**: Check for policy compliance
3. **Technical Review**: Verify app functionality
4. **Approval**: App goes live

### Common Rejection Reasons:
- ❌ Privacy policy not accessible
- ❌ Data Safety form incomplete
- ❌ Screenshots don't match app
- ❌ Misleading description
- ❌ Policy violations
